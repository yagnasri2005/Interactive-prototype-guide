# Wireframe Checklist

Use this checklist to ensure your wireframes are complete and ready for prototyping.

## Content & Information
- [ ] All key screens identified and created
- [ ] Information hierarchy is clear (important content prominent)
- [ ] All required content elements included
- [ ] Placeholder text or labels for all content areas
- [ ] Edge cases considered (empty states, errors, loading)

## Layout & Structure
- [ ] Consistent grid system applied
- [ ] Proper spacing and alignment
- [ ] Navigation elements clearly visible
- [ ] Visual hierarchy established through size/placement
- [ ] Responsive design considered (mobile/tablet/desktop)

## Interactive Elements
- [ ] All buttons labeled clearly
- [ ] Form fields and labels defined
- [ ] Links and navigation paths identified
- [ ] Interactive states documented (hover, active, disabled)
- [ ] Call-to-action (CTA) elements prominent

## Flows & Navigation
- [ ] User flows mapped between screens
- [ ] Happy path clearly defined
- [ ] Alternative paths documented
- [ ] Entry and exit points clear
- [ ] Navigation menu structure defined

## Annotations & Documentation
- [ ] All interactive elements annotated
- [ ] User actions and expected outcomes noted
- [ ] Business logic documented
- [ ] Conditional flows explained
- [ ] Links between screens indicated

## Stakeholder Review
- [ ] Wireframes reviewed by project team
- [ ] Feedback from stakeholders incorporated
- [ ] Any blockers or dependencies identified
- [ ] Sign-off obtained before moving to prototype

## Preparation for Prototyping
- [ ] Component names standardized
- [ ] Screen naming convention established
- [ ] Colors/typography guidelines defined
- [ ] Asset requirements identified
- [ ] Prototype scope clarified

---

**Once all items are checked, you're ready to move to Phase 2: Prototyping!**
