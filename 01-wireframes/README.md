# Phase 1: Wireframing Fundamentals

Wireframes are the blueprint for your interactive prototype. They define structure, hierarchy, and user flow without visual complexity.

---

## What is a Wireframe?

A wireframe is a low-fidelity visual representation of a user interface that:
- Shows layout and structure
- Establishes information hierarchy
- Maps user flows and interactions
- Uses basic shapes and text (no colors, images, or visual design)
- Serves as a communication tool between designers and stakeholders

---

## Types of Wireframes

### 1. Sketched Wireframes
- Hand-drawn on paper or digital tablet
- Quick and iterative
- Best for brainstorming
- **Time:** 15-30 minutes

### 2. Low-Fidelity Wireframes
- Simple gray boxes and text
- Created in Figma, Balsamiq, or similar tools
- Used for concept validation
- **Time:** 1-2 hours

### 3. High-Fidelity Wireframes
- Detailed layouts with precise spacing
- Includes typography and color
- Almost looks like a finished design
- **Time:** 4-8 hours

---

## Wireframing Process

### Step 1: User Research
- Understand user needs and goals
- Map user journeys
- Identify key tasks
- **Deliverable:** User personas and journey maps

### Step 2: Information Architecture
- Define content hierarchy
- Organize information logically
- Plan navigation structure
- **Deliverable:** Site map or information architecture diagram

### Step 3: Create Wireframes
- Sketch key screens
- Show content placement
- Indicate interactive elements
- Include annotations and notes
- **Deliverable:** Wireframe document (5-10 key screens)

### Step 4: Define Flows
- Map user paths through screens
- Identify decision points
- Plan state changes
- **Deliverable:** User flow diagram

### Step 5: Validate & Iterate
- Get stakeholder feedback
- Refine based on input
- Prepare for prototyping
- **Deliverable:** Finalized wireframes

---

## Wireframing Best Practices

✅ **Do:**
- Use consistent grid systems
- Keep it simple and focused
- Label interactive elements
- Include annotations explaining logic
- Test flows with stakeholders early
- Use standard UI patterns

❌ **Don't:**
- Add visual design (colors, images, detailed styling)
- Include final copy—use "Lorem Ipsum"
- Make wireframes too high-fidelity
- Forget to include edge cases
- Design in isolation—involve stakeholders

## Key Wireframing Principles

### 1. Information Hierarchy
Most important content at the top, secondary content below.

### 2. Consistency
Use same patterns for similar elements across screens.

### 3. Simplicity
Remove unnecessary elements—focus on core functionality.

### 4. Clarity
Make it obvious what each element does.

### 5. User-Centered
Design based on actual user needs and behaviors.

---

## Common Wireframe Layouts

### Navigation Pattern
- Top navigation bar with main menu items
- Consistent across all screens

### Hero + Content
- Large hero image/section
- Below: featured content
- Sidebar: secondary information

### Grid Layout
- Content in responsive grid
- Flexible for different screen sizes
- Works well for product listings

### Form Layouts
- Single or multi-column forms
- Clear labels and input fields
- Submit and cancel buttons
- Validation message areas
