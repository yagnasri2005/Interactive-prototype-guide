# Phase 3: User Testing

Conduct effective user testing sessions to validate your prototype and gather actionable feedback.

---

## Why Test with Users?

✅ **Validate Assumptions**
- Test what actually works vs. what we think works
- Identify unexpected user behaviors

✅ **Find Usability Issues**
- Discover navigation problems
- Identify confusing interactions
- Reveal accessibility issues

✅ **Gather Qualitative Insights**
- Understand user motivations
- Learn about user frustrations
- Get detailed feedback

✅ **Make Data-Driven Decisions**
- Prioritize improvements
- Justify design choices
- Convince stakeholders

---

## Testing Methodologies

### Moderated Testing
- **Format:** In-person or remote with facilitator
- **Duration:** 30-60 minutes per session
- **Participants:** 5-8 users
- **Cost:** Medium
- **Best for:** Detailed feedback, follow-up questions
- **Documentation:** [Moderated Testing Guide](./testing-methodologies.md)

### Unmoderated Testing
- **Format:** Self-directed, recorded
- **Duration:** 10-20 minutes per session
- **Participants:** 10-20+ users
- **Cost:** Low to medium
- **Best for:** Quick feedback, scale
- **Documentation:** [Unmoderated Testing Guide](./testing-methodologies.md)

### A/B Testing
- **Format:** Compare two design variations
- **Duration:** Ongoing
- **Participants:** 50+
- **Cost:** Medium to high
- **Best for:** Validation, optimization

### Card Sorting
- **Format:** Users organize content
- **Duration:** 15-30 minutes
- **Participants:** 15-30 users
- **Cost:** Low to medium
- **Best for:** Information architecture

## Testing Process

### 1. Planning (1-2 hours)
- Define testing objectives
- Create test plan
- Recruit participants
- **Output:** [Test Plan](./templates/test-plan-template.md)

### 2. Preparation (2-3 hours)
- Write test script
- Create tasks
- Prepare observation forms
- Set up recording
- **Output:** [Session Script](./templates/session-script.md)

### 3. Participant Recruitment (Varies)
- Screener questions
- Incentive structure
- Scheduling
- **Output:** [Screener Questionnaire](./templates/participant-screener.md)

### 4. Testing Sessions (Sessions)
- Conduct 5-8 moderated tests OR 10+ unmoderated tests
- Observe user behavior
- Take detailed notes
- Record sessions (with permission)
- **Output:** [Observation Notes](./templates/observation-notes-template.md)

### 5. Analysis (4-6 hours)
- Identify patterns
- Categorize feedback
- Synthesize insights
- Prioritize findings
- **Output:** Feedback report and insight matrix

---

## Key Metrics

### Quantitative Metrics
- **Task Success Rate:** % of users who completed task
- **Time on Task:** How long task took
- **Error Rate:** # of mistakes users made
- **System Usability Scale (SUS):** Overall usability score (0-100)

### Qualitative Insights
- User frustrations
- Unclear navigation
- Missing features
- Unexpected use cases
- Feature requests

---

## Best Practices

✅ **Do:**
- Test with 5-8 users minimum
- Recruit diverse participants
- Use realistic tasks
- Let users talk without interrupting
- Record all sessions (with permission)
- Take detailed notes
- Test early and often
- Use findings to prioritize improvements

❌ **Don't:**
- Test with only friends/family
- Ask leading questions
- Interrupt user attempts
- Ignore qualitative feedback
- Test without clear objectives
- Blame users for issues
- Make changes without testing
