# User Testing Methodologies

Detailed guide to different testing approaches and when to use each.

---

## Moderated Testing (Recommended for First Tests)

### Overview
Facilitator guides user through tasks while observing interactions and gathering feedback.

### Setup
- **Time:** 45-60 minutes per session
- **Format:** Video call (Zoom, Google Meet) or in-person
- **Participants:** 5-8 users
- **Facilitator:** 1 person
- **Observer:** Optional 2nd person

### Process
1. Warm-up conversation (5 min)
2. Background questions (5 min)
3. Task completion (30 min)
4. Wrap-up and questions (10 min)
5. Thank participant and send incentive

### Pros
- Deep qualitative insights
- Can ask follow-up questions
- Understand user reasoning
- Catch unexpected behaviors

### Cons
- Time-intensive
- Smaller sample size
- Facilitator bias possible
- More expensive

---

## Unmoderated Testing (For Scale & Quick Feedback)

### Overview
Users complete tasks independently, with session automatically recorded.

### Setup
- **Time:** 10-20 minutes per session
- **Platform:** Maze, Lookback, UserTesting.com
- **Participants:** 10-30 users
- **Facilitator:** None (automated)

### Process
1. User accesses test link
2. Answers screening questions
3. Watches intro video
4. Completes tasks
5. Answers post-test questions

### Pros
- Quick to run
- Larger sample size
- Geographically diverse
- Less expensive
- No facilitator bias

### Cons
- Less detailed feedback
- Can't ask follow-ups
- Higher drop-off rate
- Misunderstandings possible

---

## Card Sorting

### Overview
Users organize content categories to understand information architecture.

### Setup
- **Time:** 15-30 minutes
- **Participants:** 15-30 users
- **Method:** Open or Closed

### Open Card Sorting
- Users create their own categories
- Best for: Discovering new organization
- Cost: Medium

### Closed Card Sorting
- Users sort into predefined categories
- Best for: Validating taxonomy
- Cost: Low

### Pros
- Quick to run
- Valuable for IA validation
- Large sample size possible
- Remote-friendly

### Cons
- Limited to organization testing
- No context about usage
- Doesn't test full flow

---

## A/B Testing

### Overview
Compare two design variations to determine which performs better.

### Setup
- **Time:** Ongoing (days/weeks)
- **Participants:** 50-500+
- **Variations:** 2-3 designs

### Process
1. Define success metric
2. Create variations
3. Deploy to real users
4. Measure performance
5. Analyze results

### Pros
- Real behavior data
- Statistically valid
- No bias
- At-scale testing

### Cons
- Requires live traffic
- Takes time
- Need technical setup
- Doesn't explain "why"

---

## Tree Testing

### Overview
Test information architecture using a text-based navigation structure.

### Setup
- **Time:** 10-15 minutes
- **Participants:** 10-20 users
- **Platform:** Optimal Workshop, similar tools

### Pros
- Quick IA validation
- Focus on structure
- Easy to iterate

### Cons
- Doesn't test full design
- No visual feedback
- Limited context

---

## Five-Second Test

### Overview
Users view design for 5 seconds and describe first impressions.

### Setup
- **Time:** 2-5 minutes
- **Participants:** 10-20 users
- **Method:** Image shown briefly

### Pros
- Quick feedback
- Tests first impression
- Easy to run
- Inexpensive

### Cons
- Very limited feedback
- No interaction testing
- Doesn't test usability

---

## Choosing a Methodology

| Goal | Methodology | When to Use |
|------|------------|-------------|
| Validate flows | Moderated Testing | Early concepts |
| Quick feedback | Unmoderated Testing | Rapid iteration |
| Test IA | Card Sorting | Before prototyping |
| Find winner | A/B Testing | Multiple options |
| Test navigation | Tree Testing | Structure validation |
| First impressions | Five-Second Test | Visual communication |

---

**Start with moderated testing for your first round—it provides the most detailed insights!**
