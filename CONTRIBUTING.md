# Contributing to Interactive Prototype Guide

We welcome contributions from designers, researchers, and anyone passionate about prototyping and user testing!

---

## How to Contribute

### 1. Report Issues
- Found an error or unclear section?
- Have a suggestion for improvement?
- [Open an issue](https://github.com/yagnasri2005/Interactive-prototype-guide/issues)

### 2. Add Resources
- Have a great article or tutorial?
- Know a useful tool?
- Add to [resources/](./resources/)

### 3. Share Case Studies
- Have a project to highlight?
- Want to share lessons learned?
- Add to [case-studies/](./case-studies/)

### 4. Improve Documentation
- See a typo?
- Want to expand a section?
- [Open a pull request](https://github.com/yagnasri2005/Interactive-prototype-guide/pulls)

### 5. Add Templates
- Created a useful template?
- Have a better version of existing templates?
- Submit to relevant [templates/](./templates/) folder

---

## Contribution Guidelines

### Before You Start
1. Check if your contribution aligns with the guide's scope
2. Review existing content to avoid duplication
3. Ensure quality and accuracy

### Content Standards
- Clear, concise writing
- Well-organized sections
- Examples and templates where helpful
- Links to relevant resources
- Proper markdown formatting

### File Naming
- Use kebab-case: `my-guide.md`
- Be descriptive: `card-sorting-methodology.md`
- Avoid generic names: not `guide.md`

### Pull Request Process
1. Fork the repository
2. Create a feature branch: `git checkout -b add-my-contribution`
3. Make your changes
4. Commit with clear message: `git commit -m "Add [feature] to [section]"`
5. Push to your fork
6. Open a PR with description of changes
